# SECP 3223: Data Analytics Programming

## 📌 Course Overview
* **School/Faculty:** Faculty of Computing
* **Program Name:** Bachelor of Computer Science (Data Engineering)
* **Course Code:** SCST 2113
* **Academic Session/Semester:** 2024/2025-2
* **Credit Hours:** 3

---

## 📖 Course Synopsis
This course introduces the use of any available open-source programming language specifically for Data Science (current trend is Python). Students will learn about powerful ways to store and manipulate data to do data analysis. The course is divided into two parts. In Part 1, students will learn general programming practices and tools. Part 2 will focus more on data analysis, studying statistical techniques, machine learning and presentation of findings.

---

## 💭 Reflection

### What I Learned
Taking SCST 2113 provided me with the programming tools required to execute end to end data science operations. By utilizing Python and its data engineering libraries, I developed a strong foundation in structural data processing. I progressed from managing fundamental built in data structures to masterfully cleaning datasets, executing data aggregation loops, and implementing exploratory data visualizations to turn unorganized variables into clear business stories.

### Challenges & Growth
The most demanding phase of the course was transforming unstructured files into analytical pipelines using structured engineering methodologies. Moving beyond basic scripting to implement multi step data wrangling routines and introductory machine learning algorithms required a strict focus on data integrity. Overcoming syntax errors and data alignment issues during hands on labs forced me to think critically about how memory, array structures, and relational abstractions operate in data processing pipelines.

### Future Application
The scripting and architectural concepts learned throughout this curriculum serve as the core technical baseline for my journey as a computer science professional. I will directly apply these Python scripting practices to automate Extract, Transform, Load (ETL) cycles, build repeatable preprocessing systems, and implement baseline statistical learning estimators. This practical training ensures I can confidently approach raw, messy data environments and shape them into stable, production ready assets.
